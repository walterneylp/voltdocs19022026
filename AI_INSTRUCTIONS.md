# AI Instructions — VoltDocs

## Branches
- main: estável (somente via merge)
- dev: desenvolvimento diário
- feature/*, fix/*, refactor/*, hotfix/*: tarefas

## Versionamento
SemVer: v0.x.y
- x: mudança relevante / novo módulo
- y: correção / ajuste

## Commits
Use mensagens curtas:
- feat: ...
- fix: ...
- refactor: ...
- chore: ...
- docs: ...

## Regras obrigatórias
1) Antes de mudar qualquer coisa: `git status -sb` e `git branch --show-current`
2) Nunca commitar direto no main
3) Mostrar `git diff` antes do commit
4) Commits pequenos e frequentes
5) Ao final: `git status`, `git log -1`, `git push`
